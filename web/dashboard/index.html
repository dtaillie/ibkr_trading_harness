Pick the page by the question you are answering

Use these shortcuts when the dashboard feels dense. Start with current health, saved data, or replay setup, then drill into runs and operations only when something needs inspection.

Is it running?Overview shows current health, mode, latest events, and open positions. How is it doing?Performance shows equity, returns, drawdown, rollups, and trade rows. What data do I have?Data Library shows roots, symbols, recommended files, charts, and diagnostics. Can I simulate this?Workbench turns saved data into validated replay or simulated-paper drafts. What exactly happened?Runs exposes decisions, orders, fills, rejects, artifacts, and logs. What is broken?Operations covers diagnostics, remote nodes, supervisors, commands, and cleanup.

First Run Checklist
  1. Confirm Gateway/API status in Overview.
  2. Open Data Library and verify your real data roots are visible.
  3. Inspect a saved dataset before using it in a replay.
  4. Build or load a draft in Workbench.
  5. Run replay or simulated paper, then inspect Performance and Runs.
Today's Strategy Performance
  1. Start in Overview for current mode, latest signal/fill, and open-position cards.
  2. Open Performance for period-filtered equity, drawdown, daily returns, and fill-derived trade rows.
  3. Open Runs when a number looks suspicious and inspect the exact artifact or log.
  4. Check Operations if the latest run is stale, the API is disconnected, or no current telemetry exists.
Common Workflows
Check a live paper day Overview shows whether a runner is publishing telemetry, Performance shows the latest equity/account window, and Runs shows the exact decisions, orders, fills, rejects, and artifacts behind the number.
Review a completed fetch Open Fetch Jobs, inspect the job, use Show Outputs in Data Library, then inspect or compare the saved files that were produced under configured roots.
Use fetched data in replay Inspect the saved file first, confirm quality and timestamp coverage, select it in Workbench, preview alignment, generate a draft, validate it, run replay, then open results in Performance.
Find missing symbols Use Data Library root cards, Storage Audit, and Find Missing Symbol to distinguish missing files from unconfigured roots, parser skips, catalog caps, and fetch no-data responses.
Data To Simulation Fast Path
1. Confirm roots Open Data Library and use Data Source Map to see configured, suggested, hidden, capped, parser-error, and unavailable roots before searching symbols.
2. Pick files Use Data Home shortlist, Symbol Browser, or Symbol Directory to inspect and compare saved files before sending them to Workbench.
3. Recover fetches If a produced file is missing, open Fetch Jobs and use Fetch Recovery Plan to decide whether to fix permissions, contracts, data roots, no-data rows, or resume failed work.
4. Simulate Open Workbench Home first. It summarizes selected data, alignment, draft validation, latest run, loaded artifacts, and the next action before the full form.
Workflow Shortcuts
Data Detail to Workbench Open any saved file, review quality and range, then use Use In Workbench to carry that file and viewer date window into Config Builder.
Compare to Workbench After comparing saved datasets, use Use In Workbench to send the selected comparison set and date window into simulation setup.
Fetch to Workbench Inspect a fetch manifest, confirm outputs are Data Library-visible, then use Use Outputs In Workbench to replay the produced files.
Workbench to Results After running a draft, use Workbench Run Result or the Results buttons to open Performance and Runs without searching artifact tables.
Inspect Saved Historical Data
  1. Open Data Library and set the catalog limit high enough for your root.
  2. Use search, asset, source, bar-size, and quality filters to narrow the table.
  3. Click Inspect on a dataset row.
  4. Use the Data Detail date range, point count, and sampled/full controls for offline chart inspection.
  5. Review gaps, nulls, timezone, row count, and quality before using the file in Workbench.
Why Only A Few Symbols?
  1. Check Configured Roots and suggested roots in Data Library.
  2. Increase the catalog limit if the root has more files than the current scan limit.
  3. Use Find Missing Symbol to see whether the file is visible, malformed, scan-limited, or outside configured roots.
  4. Add real cache/history roots with dashboard.data_roots in a private local config.
Public Publishing Boundary
  • Keep private strategy plugins and tuned configs out of the public repo.
  • Keep account IDs, logs, credentials, and live/paper configs ignored locally.
  • Publish example configs only when they are no-edge and operationally illustrative.
  • Run the public readiness audit before pushing.
Glossary
Runner
A local process that evaluates a plugin and records decisions, orders, fills, and account snapshots.
Draft
A generated example runner config saved by the Workbench.
Replay
Run a plugin against saved bars.
Shadow
Observe decisions without submitting orders.
Simulated paper
Use local simulated fills and account state.
Paper
Submit to a broker paper account.
Fill
An executed order event from simulated or broker paper execution.
Reject
An order or intent blocked by validation, risk guards, broker response, or missing data.
Artifact
Saved summary, decisions, orders, fills, and account snapshots.
Data root
A configured local directory that the dashboard scans for CSV/parquet files.
Stale bar
Latest market data is older than expected.
